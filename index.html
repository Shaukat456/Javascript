14. Script Tags - Defer vs Async

Use for: Controlling how and when JavaScript files are loaded and executed

Normal Script Loading (Default)

<script src="script.js"></script>

Behavior: HTML parsing stops → Script downloads → Script executes → HTML parsing resumes

Use when: Script is critical and must run immediately

Async Script Loading

<script src="script.js" async></script>

Behavior: HTML parsing continues while script downloads → Script executes immediately when ready (may interrupt parsing)

Use when: Script is independent and doesn't rely on DOM or other scripts (e.g., analytics, ads)

Order: Scripts may execute in any order, not necessarily in the order they appear

Defer Script Loading

<script src="script.js" defer></script>

Behavior: HTML parsing continues while script downloads → Script executes only after HTML parsing is complete

Use when: Script needs the full DOM to be ready and should maintain execution order

Order: Scripts execute in the order they appear in the document

Practical Examples

<!-- In the <head> section -->
<script src="analytics.js" async></script> <!-- Independent script -->
<script src="jquery.js" defer></script> <!-- Library needed by other scripts -->
<script src="main.js" defer></script> <!-- Main application script -->

<!-- At the end of <body> (alternative to defer) -->
<script src="dom-manipulation.js"></script> <!-- Runs after DOM is ready -->

Visual Timeline Comparison

Normal:

HTML Parsing ████████ (stops) → Script Download + Execute ████ → HTML Parsing ████████

Async:

HTML Parsing ████████████████████████████ (continues)

Script Download ████████ → Execute ██ (interrupts parsing)

Defer:

HTML Parsing ████████████████████████████ (completes)

Script Download ████████ → Execute ██ (after parsing)

Decision Guide

  • Use async for:
    • Analytics scripts (Google Analytics, tracking pixels)
    • Advertisement scripts
    • Social media widgets
    • Scripts that don't depend on DOM or other scripts
  • Use defer for:
    • Scripts that manipulate the DOM
    • Scripts that depend on other deferred scripts
    • Main application JavaScript
    • jQuery and other libraries
  • Use normal loading for:
    • Critical scripts needed immediately
    • Scripts that other inline scripts depend on
    • Small, essential scripts

Important Notes

  • defer and async only work with external scripts (src attribute)
  • If both defer and async are present, async takes precedence
  • Inline scripts (without src) always block HTML parsing
  • Scripts in the <head> with defer behave like scripts at the end of <body>

15. HTML Validation, Accessibility & Performance

HTML Validation

Why validate: Ensures cross-browser compatibility and professional quality

Validation tools:

  • W3C Markup Validator: https://validator.w3.org/
  • HTML5 Validator: https://html5.validator.nu/
  • Browser DevTools: Built-in error reporting
  • VS Code Extensions: Real-time validation

Common validation errors to avoid:

  • Unclosed tags: <p>Text <strong>bold</p> ❌
  • Improper nesting: <p><div>content</div></p> ❌
  • Missing alt attributes on images ❌
  • Duplicate IDs on same page ❌
  • Missing DOCTYPE declaration ❌

Accessibility (A11Y) Best Practices

Why accessibility matters: Inclusive design, legal compliance, better SEO

Essential accessibility practices:

1. Use Semantic HTML


<button>Click me</button>
<nav><ul><li><a href="#">Home</a></li></ul></nav>


<div onclick="doSomething()">Click me</div>
<div class="nav"><div><div><span>Home</span></div></div></div>

2. Proper Alt Text

<img src="chart.png" alt="Sales increased 25% from Q1 to Q2 2025"> ✅
<img src="chart.png" alt="chart"> ❌
<img src="decorative.png" alt=""> ✅ (decorative image)

3. Form Labels and Instructions

<label for="email">Email Address (required)</label>
<input type="email" id="email" name="email" required aria-describedby="email-help">
<div id="email-help">We'll never share your email</div>

<h1>Page Title</h1>
  <h2>Section</h2>
    <h3>Subsection</h3>

5. Color and Contrast

  • Minimum contrast ratio: 4.5:1 for normal text
  • Minimum contrast ratio: 3:1 for large text
  • Don't rely on color alone to convey information
  • Test with tools like WebAIM Contrast Checker

6. Keyboard Navigation

  • All interactive elements must be keyboard accessible
  • Visible focus indicators
  • Logical tab order
  • Skip links for main content

Performance Best Practices

Why performance matters: User experience, SEO rankings, conversion rates

1. Optimize Images

  • Use appropriate formats (WebP, AVIF for modern browsers)
  • Compress images before uploading
  • Use responsive images with srcset
  • Lazy load images below the fold
<img src="small.jpg" srcset="small.jpg 300w, medium.jpg 600w, large.jpg 1200w" sizes="(max-width: 600px) 300px, (max-width: 1200px) 600px, 1200px" alt="Responsive image" loading="lazy">

2. Minimize HTTP Requests

  • Combine CSS and JavaScript files
  • Use CSS sprites for small images
  • Inline critical CSS
  • Use icon fonts or SVG sprites

3. Optimize Loading

  • Use defer/async for scripts
  • Preload critical resources
  • Minimize render-blocking resources
  • Use compression (gzip/brotli)

4. Measure Performance

  • Google PageSpeed Insights: Performance scoring
  • Lighthouse: Built into Chrome DevTools
  • WebPageTest: Detailed performance analysis
  • GTmetrix: Performance monitoring

16. HTML Best Practices & Professional Tips

Fundamental Principles

  • Semantic HTML first: Choose tags based on meaning, not appearance
    Use <button> for buttons, <nav> for navigation, <article> for articles
  • Accessibility is not optional: Always include alt text, proper labels, and keyboard navigation
    Your website should work for everyone, including users with disabilities
  • Mobile-first approach: Design for mobile devices first, then enhance for desktop
    Most web traffic comes from mobile devices
  • Progressive enhancement: Start with basic HTML, then enhance with CSS and JavaScript
    Your content should be accessible even if CSS/JS fails to load

Code Organization

1. Document Structure

<!DOCTYPE html>
<html lang="en">
<head>
  <!-- Meta tags first -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- SEO meta tags -->
  <title>Page Title</title>
  <meta name="description" content="...">
  
  <!-- External resources -->
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <!-- Content here -->
  <!-- Scripts at the end -->
  <script src="script.js"></script>
</body>
</html>

2. Consistent Indentation and Formatting

  • Use 2 or 4 spaces consistently (not tabs)
  • Lowercase tag and attribute names
  • Quote all attribute values
  • One attribute per line for complex elements

3. Meaningful Comments

<!-- Navigation Section -->
<nav role="navigation" aria-label="Main navigation">
  <!-- TODO: Add search functionality -->
</nav>

<!-- Main Content Area -->
<main>
  <!-- Hero section with call-to-action -->
</main>
