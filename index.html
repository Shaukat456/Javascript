13. Script Tags - Defer vs Async

Use for: Controlling how and when JavaScript files are loaded and executed

Normal Script Loading (Default)

<script src="script.js"></script>

Behavior: HTML parsing stops → Script downloads → Script executes → HTML parsing resumes

Use when: Script is critical and must run immediately

Async Script Loading

<script src="script.js" async></script>

Behavior: HTML parsing continues while script downloads → Script executes immediately when ready (may interrupt parsing)

Use when: Script is independent and doesn't rely on DOM or other scripts (e.g., analytics, ads)

Order: Scripts may execute in any order, not necessarily in the order they appear

Defer Script Loading

<script src="script.js" defer></script>

Behavior: HTML parsing continues while script downloads → Script executes only after HTML parsing is complete

Use when: Script needs the full DOM to be ready and should maintain execution order

Order: Scripts execute in the order they appear in the document

Practical Examples

<!-- In the <head> section -->
<script src="analytics.js" async></script> <!-- Independent script -->
<script src="jquery.js" defer></script> <!-- Library needed by other scripts -->
<script src="main.js" defer></script> <!-- Main application script -->

<!-- At the end of <body> (alternative to defer) -->
<script src="dom-manipulation.js"></script> <!-- Runs after DOM is ready -->

Visual Timeline Comparison

Normal:

HTML Parsing ████████ (stops) → Script Download + Execute ████ → HTML Parsing ████████

Async:

HTML Parsing ████████████████████████████ (continues)

Script Download ████████ → Execute ██ (interrupts parsing)

Defer:

HTML Parsing ████████████████████████████ (completes)

Script Download ████████ → Execute ██ (after parsing)

Decision Guide

  • Use async for:
    • Analytics scripts (Google Analytics, tracking pixels)
    • Advertisement scripts
    • Social media widgets
    • Scripts that don't depend on DOM or other scripts
  • Use defer for:
    • Scripts that manipulate the DOM
    • Scripts that depend on other deferred scripts
    • Main application JavaScript
    • jQuery and other libraries
  • Use normal loading for:
    • Critical scripts needed immediately
    • Scripts that other inline scripts depend on
    • Small, essential scripts

Important Notes

  • defer and async only work with external scripts (src attribute)
  • If both defer and async are present, async takes precedence
  • Inline scripts (without src) always block HTML parsing
  • Scripts in the <head> with defer behave like scripts at the end of <body>

14. HTML Best Practices

  • Always use semantic HTML: Choose tags based on meaning, not appearance
  • Include alt attributes: Always provide alt text for images
  • Use proper nesting: Ensure tags are properly opened and closed
  • Validate your HTML: Use tools like W3C Markup Validator
  • Keep it accessible: Use proper heading hierarchy, labels for form elements
  • Use lowercase: Write all HTML tags and attributes in lowercase
  • Quote attribute values: Always use quotes around attribute values
  • Close self-closing tags: Use proper syntax for void elements
